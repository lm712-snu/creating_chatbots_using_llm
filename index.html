Causal Attention
Mask

3Blue1Brown (Ch.6): "During training, the model predicts every possible next token in parallel for efficiency." But to avoid cheating — where position i can see future tokens — we apply a causal mask. Umar Jamil (t=42:24): future scores are set to −∞ so exp(−∞)=0 after softmax. The result is the lower-triangular pattern.

Cross-Attention

Early Fusion

Joint Self-Attention

Concatenate all tokens: [TEXT; IMAGE; AUDIO]. Run full self-attention over entire sequence. Every token attends every other regardless of modality. Used in GPT-4V, Gemini.

X=[x_text;x_img;x_aud]
H=Transformer(X)

Cross-Attention Fusion

Gated Modality Bridge

Separate encoder per modality produces context vectors. Decoder queries text while cross-attending over image/audio context. Efficient — image encoder runs once. Used in BLIP-2, LLaVA.

C_img=VisionEncoder(I)
H_t=CrossAttn(Q=h_text,K=V=C_img)
// Chapter IV

The Output
Probability Distribution

3Blue1Brown Ch.5: "After many iterations, all the information necessary to predict the next word needs to be encoded into the last vector of the sequence, which will go through one final computation to produce a probability distribution over all possible chunks of text that might come next." This is the unembedding matrix W_U followed by softmax.
3B1B Ch.5 — UNEMBEDDING + SOFTMAX // Step 1: unembedding matrix maps last vector to logits
logits = W_U · h_T   // W_U ∈ ℝ^(|V|×d), ~617M params in GPT-3
// Step 2: softmax normalises logits to probability distribution
P(t_n|t_{<n}) = softmax(logits/τ) = exp(logits/τ) / Σ exp(...)

// Multimodal — all modalities in context window:
P(t_n|t_{<n},I,A) = softmax(W_U·Transformer([text;img;aud])[-1])

3B1B Ch.5: "The machine learning term for the unnormalized inputs is logits." Raw scores from W_U — context: "Harry Potter...least favorite Professor" → model scores Snape highest.

Prior Knowledge
P(t_n)

3B1B Ch.5: Common words like "the" and "a" appear far more often than rare words. The embedding matrix W_E and training data encode this frequency information implicitly.

Contextual Evidence
h_T encodes context

3B1B Ch.5: "The goal is to enable each vector to soak up meaning that is more rich and specific." The final hidden state h_T carries all context — text + image + audio — compressed into d_model dimensions.

Posterior Distribution
P(t_n | context)

3B1B Ch.5: "A well-trained network would presumably assign a high number to the word Snape." The posterior is context-sensitive — the same logit position gets very different probabilities depending on what came before.

Normalisation
Σ exp(ℓ_k) = Z

3B1B Ch.5: "The way to get a valid probability distribution is the softmax function." The denominator Z = Σexp(ℓ_k) is the normalising constant — guarantees all probabilities sum to exactly 1.0.

// Chapter V

Decoding
Strategies

3Blue1Brown Ch.5: "Once you have a prediction model, one simple way to make it generate longer text is to give it an initial bit of text to work with, have it predict the next word, take a random sample from the distribution it just generated, then run it all again." The sampling strategy is the key decision — how do we pick from the probability distribution at each step?
1

Greedy Decoding

Always Pick the Mode

Select the single highest-probability token every step — equivalent to temperature τ→0. Fast and deterministic. 3B1B Ch.5: "The largest logit gets the most probability." Greedy always picks that token. Problem: repetitive, generic output — misses coherent but lower-probability continuations.

t* = argmaxt p(t|context) = argmaxt logit_t
2

Beam Search

Keep B Best Partial Sequences

Umar Jamil: used extensively in machine translation. Maintains B candidate sequences simultaneously. At each step, expands all beams and keeps the top B by log-probability sum. B=4 or B=5 typical. Better than greedy but still can produce safe, generic outputs. Not used in ChatGPT-style generation.

score(t₁…tₙ) = Σᵢ log P(tᵢ | t₁…tᵢ₋₁)
3

Temperature Sampling

Reshape Before Sampling

3Blue1Brown Ch.5: softmax with temperature τ. τ=1: sample from the raw model distribution — "repeated prediction and sampling is what ChatGPT does." τ<1: sharper, more confident. τ>1: flatter, more surprising. The "temperature" knob controls how literally the model interprets its own probability distribution.

p_τ(t) = exp(logit_t / τ) / Σ exp(logit_k / τ)
4

Top-P Nucleus Sampling

Smallest Set with ≥ p Mass

Select the smallest vocabulary subset whose cumulative probability ≥ p (typically 0.9). Adapts dynamically — when the model is confident about "Snape" (3B1B example), only a few tokens qualify; when uncertain, many more are included. More principled than a fixed top-K cutoff.

V_p = min V' s.t. Σ_{t∈V'} p(t) ≥ 0.9
5

Top-K Sampling

Fixed K-Token Cutoff

Restrict to the K highest-probability tokens, renormalise, then sample. K=50 is typical. Simple and effective. Drawback: fixed cutoff can be too broad when the model is confident, or too narrow when it is uncertain — unlike Top-P which adapts automatically.

p'(t) = p(t)/Z · 𝟙[t ∈ top-K(p)], Z = Σ_{top-K} p(t)

KV-Cache

Efficient Repeated Prediction

3B1B Ch.5: "repeated prediction and sampling" means the forward pass runs again for every new token. Without a cache: O(T²) per step — recomputing all past attention. KV-Cache stores computed K and V matrices for all past positions and layers, reducing each new step to O(T).

cache: K₁…K_{n-1}, V₁…V_{n-1} per layer

Speculative Decoding

Draft Fast, Verify in Parallel

A small fast model ("draft model") proposes k tokens speculatively. The large model verifies all k positions in a single parallel forward pass — since transformer attention processes all positions simultaneously. Accepted tokens are kept; first rejected token triggers resampling. 2–3× wall-clock speedup with mathematically identical output distribution.

Accept t if U[0,1] < p_large(t) / p_draft(t)
